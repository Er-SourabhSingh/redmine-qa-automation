# BUG-RDV-020 — GitLab commit SHA links use /commit/ instead of /-/commit/ URL pattern

## Summary
GitLab commit links generated by the plugin use the legacy `/commit/<sha>` URL pattern instead of the current GitLab standard `/-/commit/<sha>` pattern. GitHub commit links are generated correctly.

## Steps to Reproduce
1. Navigate to `http://localhost:3008/projects/phoenix-platform/devops_commits/31` (a GitLab-sourced commit).
2. Inspect the "Open in Repository" link `href`.

## Expected Result
- Link: `https://gitlab.com/{namespace}/{project}/-/commit/{full_sha}`

## Actual Result
- Link: `https://gitlab.com/test/repo/commit/b2c3d4e5f6a7b2c3d4e5f6a7b2c3d4e5f6a7b2c3` (missing `/-/`)

## Screenshot
No dedicated screenshot — verified via DOM inspection of commit #31 detail page.

## Notes
- The old format still redirects on gitlab.com but is deprecated and may break for self-hosted GitLab instances.
- GitHub commit links (`https://github.com/<org>/<repo>/commit/<sha>`) are generated correctly.
- Fix: append `/-/commit/` instead of `/commit/` when constructing GitLab commit URLs.
