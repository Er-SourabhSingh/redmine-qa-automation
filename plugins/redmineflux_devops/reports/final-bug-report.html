BUG-RDV-008 REST API returns DevOps data from all projects regardless of membership (IDOR) CRITICAL
TC: TC-RDV-022  |  Environment: Forge only*  |  Date: 2026-05-22  |  Role: Developer

Steps to Reproduce

  • Obtain API key for a user with membership in only one project.
  • Send GET /devops/builds.json?project_id=<other-project> with that key.
  • Repeat for commits, deployments.

Expected

HTTP 403 Forbidden for all requests to projects the user is not a member of.

Actual

HTTP 200 with full DevOps data (builds, commits) returned from projects the user has no membership in. Classic IDOR / broken access control.

Re-verification 2026-05-23 (Local): FORGE ONLY — HTTP 403 returned correctly on local for non-member users. Bug does not reproduce on Local Docker. Forge-specific; re-verify on Forge.
BUG-RDV-002 Webhook endpoint accepts payloads over 2 MB — no size limit enforced HIGH
TC: TC-RDV-003  |  Environment: Forge + Local  |  Date: 2026-05-22  |  Role: System (webhook)

Steps to Reproduce

  • Construct a valid GitHub push payload padded to >2 MB with valid HMAC-SHA256 signature.
  • POST to /devops/webhook/github/phoenix-platform.

Expected

HTTP 413 Request Entity Too Large for payloads exceeding 2 MB.

Actual

HTTP 202 Accepted. 3 MB payload processed without any size check. DoS risk via memory exhaustion.

Re-verification 2026-05-23 (Local): CONFIRMED — 3 MB payload accepted with HTTP 202.

BUG-RDV-007 Webhook background job fails with ActiveRecord::RecordInvalid after returning 202 HIGH
TC: TC-RDV-015  |  Environment: Forge + Local  |  Date: 2026-05-22  |  Role: System (webhook)

Steps to Reproduce

  • Send valid GitHub push webhook to /devops/webhook/github/phoenix-platform with minimal payload (no repository.html_url field).
  • Observe HTTP response, then check webhook event log for error_message.

Expected

HTTP 202 returned; background job completes; commit records persisted in DB.

Actual

HTTP 202 returned (correct), but background job fails: ActiveRecord::RecordInvalid: Validation failed: Repo url cannot be blank. Commit records not persisted. Root cause: background job requires repository.html_url in the push payload.

Re-verification 2026-05-23 (Local): CONFIRMED — Error confirmed in docker logs for event ID 135. Root cause identified: missing repository.html_url triggers RecordInvalid.
